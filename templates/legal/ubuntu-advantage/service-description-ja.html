Ubuntu Advantage
サービス記述書

2016年10月2日より有効

1. 目次

  1. 1.1 本書は以下のサービスを定義します。Canonicalは、本書に定義の通り、顧客契約の対象のサービスのみを提供します。
  2. 1.2 プライマリーサービス：
    • Ubuntu Advantage OpenStack Cloud Region
    • Ubuntu Advantage Server (Essential/Standard/Advanced)
    • Ubuntu Advantage Virtual Guest (Standard/Advanced)
    • Ubuntu Advantage Ceph Storage
    • Ubuntu Advantage Swift Storage
    • Ubuntu Advantage MAAS (Standard/Advanced)
    • Ubuntu Advantage Switch
    • Ubuntu Advantage Desktop (Standard/Advanced)
  3. 1.3 アドオンサービス：
    • Landscape Seats
    • Technical Account Manager
    • Dedicated Services Engineer
    • Landscape On Premises

2. サポート範囲

各サービス製品には、Canonicalのナレッジベースおよび本書の範囲内の下記のサポートへのアクセスが含まれます。ただしサポート範囲文書に記載の例外に従います。

  1. 2.1 リリース
    • Canonicalは、プロジェクトのライフサイクル中に公式ソースを使用してインストールされたUbuntuの全標準リリースのインストール、設定、保守、および管理に関するサポートを提供します。Ubuntuのバージョン別ライフサイクルについては、こちらをご覧ください：
      www.ubuntu.com/info/release-end-of-life
  2. 2.2 ハードウェア
    • Ubuntu認定ハードウェアは、当社の広範なテストおよびレビュープロセスに合格しています。Ubuntu認定プロセスの詳細および認定ハードウェアの一覧については、Ubuntu認定ページをご覧ください： www.ubuntu.com/certification
    • 本サービスは、顧客の認定ハードウェアのみに適用されます。認定されていないハードウェアのサービスを顧客から依頼された場合、Canonicalはサポートサービスを提供するために合理的な努力を払いますが、本サービス記述書に記載された義務に従わない場合があります。
  3. 2.3 パッケージ
    1. 2.3.1 本サービスは、Ubuntu MainリポジトリにあるパッケージおよびUniverse RepositoryにあるCanonical所有のパッケージのみに適用されます。ただし、(i) 「proposed」および「backports」リポジトリポケット、および (ii) 該当するサポート範囲文書に記載された除外事項は除きます。
      Universe Repositoryのサポート対象パッケージには、Jujuパッケージ、MAASパッケージ、nova-conductorパッケージ、およびこれらのパッケージと関連して使用される依存パッケージが含まれます。
    2. 2.3.2 Ubuntuアーカイブにあるバージョンから変更されているパッケージはサポート対象外です。
  4. 2.4 カーネルのライブパッチ
    1. 2.4.1顧客は、Canonicalのカーネルライブパッチデーモンの使用許諾ライセンス、利用可能なカーネルライブパッチへのアクセス、ライブパッチ機能が利用可能でUbuntu Advantageが対応しているすべてのシステムについてライブパッチのサポートを受ける権利があります。
    2. 2.4.2Canonicalでは、緊急 (Critical) および高 (High) レベルのカーネル脆弱性CVE用ライブパッチを提供します。ただし、ライブパッチシステム内の技術的限界のため、CVEの一部はライブパッチの対象外となる可能性がありますのでご注意ください。
    3. 2.4.3Canonicalでは、カーネルのサポートのための不具合修正をカーネルライブパッチとして提供することはできません。
    4. 2.4.4Ubuntu 16.04については、ジェネリックおよび低レイテンシで作動する64ビットIntel/AMD 4.4カーネル用のカーネルライブパッチが提供されています。
  5. 2.5 Landscape
    1. 2.5.1 Landscape On Premises (購入した場合) を含むすべてのLandscape製品は完全にサポートされます。
    2. 2.5.2 別途記述がない限り、Landscape SaaSシステム管理ツールへのアクセスはすべてのサポート製品に含まれています。
  6. 2.6 OpenStack
    1. 2.6.1 本セクションは、OpenStackのサポートのために購入されたサービスに適用されます。OpenStackサポートが含まれるサービスは以下の通りです：
      • Ubuntu Advantage Server Standard
      • Ubuntu Advantage Server Advanced
      • Ubuntu Advantage OpenStack Cloud Region
    2. 2.6.2 OpenStackサポートには、サポート対象ノードが5つ以上ある環境が必要です。
    3. 2.6.3 Canonical Architectureとは以下を指します：
      • Juju、MAAS、および公式チャームリリースを使用してCanonicalとの契約に基づいてデプロイされたUbuntu OpenStack環境。Canonicalとの契約に基づいてデプロイした結果、チャームのカスタマイズが必要になった場合、カスタマイズの有効期間はそのカスタマイズを含むチャームの公式リリース後90日間です。
      • CanonicalのOpenStack Autopilotを使用したUbuntu OpenStackのデプロイメント。
    4. 2.6.4 Canonical Architectureを使用しないでデプロイされた環境については、サポートはUbuntu OpenStackパッケージ自体のバグに限定されます。また、OpenStackインストールのデプロイ、設定、または問題最適化に関するサポートは含まれません。
    5. 2.6.5 デプロイされたノード1つにつき合計で最大3TBの使用可能ストレージに対するサポート。この許容量は、Ubuntu Advantage Ceph、Ubuntu Advantage Swift、またはこれらの組み合わせに対して使用できます。
    6. 2.6.6 サポート対象のOpenStackバージョン:
      • Ubuntuの長期サポート (LTS) バージョンのリリースで最初に提供されたOpenStackのバージョンは、そのUbuntuバージョンの全ライフサイクルにおいてサポートされます。
      • UbuntuのLTSバージョン後にリリースされたOpenStackのリリースは、Ubuntu Cloud Archiveで公開されます。Ubuntu Cloud Archiveにある各OpenStackリリースは、該当するOpenStackバージョンを含むUbuntuバージョンのリリース日から最低18ヶ月間、Ubuntu LTSバージョンでサポートされます。
      • OpenStackリリースのサポートスケジュールについては www.ubuntu.com/info/release-end-of-life をご覧ください。
  7. 2.7 チャーム
    1. 2.7.1 本セクションは、OpenStackのサポートのために購入されたサービスに適用されます。
    2. 2.7.2 Canonicalは、Canonicalアーキテクチャを使用してデプロイされたOpenStackクラウドを実行するために必要なチャームに対するサポートを提供します。
    3. 2.7.3 各チャームバージョンは、リリース日から1年間サポートされます。
    4. 2.7.4 チャームストアにあるサポート対象バージョンから変更されているチャームはサポートされません。

4. サポートプロセス

Canonicalは、サポートケースを解決できるよう合理的な努力を払いますが、回避策、解決策、または解決時間は保証しません。

  1. 4.1 Canonicalは、サポートプロセスに従ってサービスを提供します。

付録1 - サポート範囲

Ubuntu Advantage OpenStack Cloud Region

定義：

  • 「OpenStack」とは、CanonicaによりUbuntuに配布されている「OpenStack」と呼ばれるクラウドコンピューティングソフトウェアを指します。
  • 「リージョン」とは、専用APIエンドポイントを持つ個々のOpenStack環境を指し、通常、Identity (Keystone) サービスのみを他のリージョンと共有します。単一のデータセンターには必ず1つのOpenStackリージョンが含まれています。
  • 「パブリッククラウド」とは、第三者がゲストインスタンスを作成/管理できるクラウド環境を指します。
  • 「クラウドゲスト」とは、パブリッククラウドで実行されていないUbuntu Serverのインスタンスを指します。

サポートは、1つのリージョン、および購入したUbuntu Advantage OpenStack Cloud Regionのサイズに基づく最大数のノードに限定されます。

追加でサービスに含まれるもの：

  • OpenStackの実行に必要なすべてのパッケージに対するサポート
  • クラウドゲスト向けUbuntu Advantage Server Advanced services
  • Landscape On Premises

サービスから除外されるもの：

  • OpenStackデプロイメントの実行に必要でないワークロードに対するサポート。
  • Canonical Architectureを使用しないでデプロイされたデプロイメントに対するサポート。
  • クラウドゲスト以外のゲストインスタンスに対するサポート。

それぞれのサービス (Essential、Standard、またはAdvanced) の範囲を以下に定めます。以下のアイテムは、別途規定されている場合を除き、Ubuntu Advantage Serverサービスの対象外です。

  • Ceph
  • Swift

Essential

サービスに含まれるもの：

  • サーバー/クラウドイメージマニフェストにあるパッケージに対するサポート。

サービスから除外されるもの：

  • サーバー/クラウドイメージマニフェストにないパッケージに対するサポート。
  • サーバー/クラウドイメージから提供されたパッケージに見つかったバグ以外の問題に対するサポート。
  • 電話サポートへのアクセス。

Standard

サービスに含まれるもの：

  • 無制限の数のUbuntu LXDゲストに対するUbuntu Advantage Virtual Guest Standardサポート。

サービスから除外されるもの：

  • KVMゲストに対するUbuntu Advantageサポート。
  • Ubuntu LXDゲストに対するLandscapeシステム管理ツールへのアクセス。ただしLandscape Seatsまたは Landscape On Premisesでカバーされているものを除く。

Advanced

サービスに含まれるもの：

  • 無限の数のUbuntu LXDおよびUbuntu KVMゲストに対するUbuntu Advantage Virtual Guest Standardサポート。

サービスから除外されるもの：

  • Ubuntu LXDまたはUbuntu KVMゲストゲストに対するLandscapeシステム管理ツールへのアクセス。ただしLandscape Seatsまたは Landscape On Premisesでカバーされているものを除く。

Ubuntu Advantage Virtual Guest

  • Ubuntu Serverに対してサービスが提供されるのは、物理ホスト上の仮想化された環境またはUbuntuの認定パブリッククラウドパートナーの環境でインストールおよび実行されている場合です。認定パブリッククラウドパートナーについては、Ubuntuパートナー一覧をご覧ください：
    partners.ubuntu.com/find-a-partner
  • サービス除外は、該当するUbuntu Advantage Serverサービスのサービス除外と同じです。

Ubuntu Advantage Ceph Storage

定義：

  • 「クラスタ」とは、単一の物理データセンター内にある単一のCephインストールを指します。

サポートは、すべてのストレージプールのCephに格納されている合計データ数 (ギガバイト) によって評価されます。空き領域とすべての複製およびイレージャー
コーディングオーバーヘッドは除外されます。

Jujuを使用してCephダッシュボードが環境にデプロイされている必要があります。

無制限のストレージ容量に対するUbuntu Advantage Ceph Storageを購入した顧客は、単一クラスタでのサポートに制限されます。

追加でサービスに含まれるもの：

  • Cephデプロイメントの実行に必要なすべてのパッケージに対するサポート。
  • ストレージおよび複製要件を満たすために必要なすべてのサービスに対するサ
    ポート。これには、Cephモニターなど予備 (ストレージ以外の) ノードが含まれます。

サービスから除外されるもの：

  • Cephデプロイメントの実行に必要でないワークロードに対するサポート。

Ubuntu Advantage Swift Storage

定義：

  • 「クラスタ」とは、単一の物理データセンター内にある単一のSwiftインストールを指します。

サポートは、すべてのストレージプールのSwiftに格納されている合計データ数 (ギガバイト) によって評価されます。空き領域とすべての複製およびイレー
ジャーコーディングオーバーヘッドは除外されます。

無制限のストレージ容量に対するUbuntu Advantage Swift Storageを購入した顧客は、単一クラスタでのサポートに制限されます。

追加でサービスに含まれるもの：

  • Swiftデプロイメントの実行に必要なすべてのパッケージに対するサポート。
  • ストレージおよび複製要件を満たすために必要なすべてのサービスに対するサ
    ポート。これには、Swiftモニターなど予備 (ストレージ以外の) ノードが含まれます。

サービスから除外されるもの：

  • Swiftデプロイメントの実行に必要でないワークロードに対するサポート。

Ubuntu Advantage MAAS

サービスに含まれるもの：

  • 文書化されたMAASデプロイメントアーキテクチャに従ってMAAS環境をホストするために必要なすべてのサーバーに対するサポート。
  • MAASの実行に必要なすべてのパッケージに対するサポート。
  • Canonical提供のオペレーティングシステムイメージを使用してマシンを起動できるようにするためのサポート。
  • Canonical提供でない認定オペレーティングシステムイメージをMAASイメージに変換するために必要なツールのサポート。
  • 標準の高可用性MAAS構成を適切に実装するために必要なツールのサポート。

サービスから除外されるもの：

  • MAASデプロイメントの実行に必要でないワークロード、パッケージ、および
    サービスコンポーネントに対するサポート。
  • MAASを使用してデプロイされたノードのサポート。
  • MAASデプロイメントの設計および実装詳細のサポート。

サポートされるMAASバージョン：

  • MAASのバージョンはLTSバージョンのUbuntuで、Ubuntuアーカイブにリリースされた日付から1年間サポートされます。

Ubuntu Advantage Switch

定義：

  • 「ホワイトボックススイッチ」とは、ODMメーカーが製造し、ベンダーがラベル添付および販売を行うx86 Broadcom ASICベースのスイッチを指します。
  • 「BCOS」とは、ホワイトボックススイッチからのフル装備のレイヤー2およびレイヤー3ネットワーキングソフトウェアのUbuntu最適化バージョンです。

含まれるサービス：

  • BCOSの実行に必要な特定のUbuntuバージョンのサポート。
  • BCOSソフトウェアのサポート。

含まれないサービス：

  • BCOSの実行に必要でないワークロードに対するサポート。
  • BCOSソフトウェアと共に提供されるバージョン以外のカーネルのサポート。
  • 物理ハードウェアのサポート。ハードウェアサポートはベンダーによって提供されます。
  • Landscape SaaSおよびLandscape On Premises

Ubuntu Advantage Desktop

含まれないサービス：

  • 仮想マシン
  • マシンコンテナ
  • アプリケーションコンテナ
  • デュアルブート (他のオペレーティングシステムとの共存)
  • Ubuntuでの使用が認定されていない周辺機器
  • Ubuntuの各種コミュニティフレイバー

Standard

サービスに含まれるもの：

  • SSSD、WinbindおよびNetworkManagerを使用した基本的なネットワーク認証および接続、ならびにUbuntuのメインリポジトリ内のNetworkManager関連プラグイン

サービスから除外されるもの：

  • 開発者ツール
  • Ubuntuの基本デスクトップイメージに含まれないパッケージのサポート

Advanced

サービスに含まれるもの：

  • Canonicalツールによるデスクトップのプロビジョニング

Landscape Seats

定義：

  • 「Seats」とは、仮想マシンをLandscape SaaSまたはLandscape On Premisesに登録できる能力を指します。
  • 「仮想マシン」とは、仮想化された環境にインストールされ、そこで実行されているUbuntu Serverを指します。

サービスに含まれるもの：

  • Ubuntu Advantageサービス対象のシステムで実行されているSeats。

サービスから除外されるもの：

  • Landscapeクライアントのインストールおよび実行に必要なパッケージのサポートを超えるサポート。

テクニカルアカウントマネージャー (TAM)

定義:

  • 「TAM」とは、顧客の従業員や経営陣と直接連携するためにリモートで作業を行うCanonicalサポートエンジニアを指します。

Canonicalは、サポートサービスの向上のためにTAMによるサービスを行います。TAMはサービス期間中、以下のサービスを週あたり最大10時間行います。

  • 該当するUbuntu Advantageサービスの対象のプラットフォームおよび構成に関するサポートおよびベストプラクティスアドバイスを提供します。
  • 2週に1度、互いに合意した回数、レビューコールに参加し、顧客が抱える運用上の問題に対応します。
  • TSANetまたはCanonicalの直接パートナーシップ (該当する場合) を通じてマルチベンダー問題の調整を行います。根本的な原因が特定できたら、TAMは、そのサブシステムのベンダーと共に通常のサポートプロセスを通じてケースの解決に努めます。

Canonicalは四半期に1度、顧客とのサービスレビューミーティングを開き、サービスのパフォーマンスを評価し、改善領域を特定します。

TAMは顧客のサイトを年に1度訪問し、オンサイトテクニカルレビューを行います。

TAMは、TAMの稼働時間中、サポートケースに対応します。稼働時間外は、Ubuntu Advantageサポートプロセスに従ってサポートが提供されます。

専任サービスエンジニア (DSE)

定義：

  • 「DSE」とは、単一の顧客のためにリモートでフルタイムで作業を行う専任のCanonicalサポートエンジニアを指します。

Canonicalは、サポートサービスの向上のためにDSEによるサービスを行います。DSEはサービス期間中、現地営業時間内に以下のサービスを週あたり最大40時間行います (Canonicalの休暇に関する方針によります)。

  • 該当するUbuntu Advantageサービスの対象のプラットフォームおよび構成に関するサポートおよびベストプラクティスアドバイスを提供します。
  • DSEは、担当する顧客部門からのすべてのサポート要求を最初に受け付ける窓口となります。
  • Canonicalの標準サポート応答定義および顧客のニーズに従い、サポートのエスカレーションおよび優先順位付けを管理します。
  • 定期レビューコールに参加し、顧客が抱える運用上の問題に対応します。
  • TSANetまたはCanonicalの直接パートナーシップ (該当する場合) を通じてマルチベンダー問題の調整を行います。根本的な原因が特定できたら、DSEは、そのサブシステムのベンダーと共に通常のサポートプロセスを通じてケースの解決に努めます。
  • 適切なCanonical社内トレーニングおよび開発活動に参加します (直接参加またはリモート)。

Canonicalは四半期に1度、顧客とのサービスレビューミーティングを開き、サービスのパフォーマンスを評価し、改善領域を特定します。

DSEは、DSEの稼働時間中、サポートケースに対応します。営業時間外は、Ubuntu Advantageサポートプロセスに従ってサポートが提供されます。

Landscape On Premises

定義：

  • 「Landscape On Premises」とは、顧客のハードウェアにインストールされたCanonicalのLandscapeシステム管理ソフトウェアを指します。

サービスに含まれるもの：

  • Landscape On Premisesのインスタンスを1つダウンロードおよびインストールするためのライセンス。
  • Ubuntu Advantageサービス購入の対象となった (物理または仮想) マシンに対してLandscape On Premises管理およびモニタリングサービスを使用する能力。

デプロイ方式：

  • 「クイックスタート」インストール方式を使用してインストールされている場合、Landscape On Premisesをインストールするマシンに対するサポートが含まれます。
  • マニュアルインストール方式を使用してインストールされている場合、Landscape On Premisesをインストールする最大2台のサーバーに対するサポートが含まれます。
  • Jujuを使用してデプロイする場合、「高密度デプロイ」方式がサポートされます。

サービスから除外されるもの：

  • Landscape On Premisesパッケージに関係しないもののサポート。

サポートされるLandscape On Premisesバージョン：

  • Landscape On Premisesの各リリースは、リリース日から1年間サポートされます。

2. サポート要求の提出

  1. 2.1 顧客アカウントがサポートポータルに作成されると、顧客はサポート要求をオープンできます。
  2. 2.2 別途記載がない限り、顧客は、サポートポータル経由で、または電話でサポートチームに連絡することにより、サポートケースを提出できます。
  3. 2.3 サポートケースは、単一の独立した問題または要求である必要があります。
  4. 2.4 ケースにはチケット番号が割り当てられ、自動応答が返されます。品質保証のため、メールや電話などケースに直接入力されない連絡はすべてタイムスタンプ付きでケースに記録されます。
  5. 2.5 顧客がケースを報告する場合には、Canonicalが適切なサービスレベルを決定できるよう、顧客は影響に関する陳述書を提供する必要があります。複数のサポートケースを同時に抱えている顧客は、ビジネスへの影響の重大度に従ってケースに優先順位を付けるよう求められる場合があります。
  6. 2.6 ケースの解決中、顧客はCanonicalが要求したすべての情報を提供することが期待されます。
  7. 2.7 各ケースのレコードは、Canonicalのサポートポータル内に保持されます。顧客は現行のすべてのケースの追跡と応答が可能で、これまでのケースをレビューできます。

3. サポートの重大度レベル

  1. 3.1 サポート要求がオープンすると、Canonicalのサポートエンジニアがケース情報を検証して重大度レベルを決定し、顧客と共にケースの緊急性を評価します。
  2. 3.2 応答時間は、該当するサービスのサービス記述書に規定された通りです。
  3. 3.3 重大度レベルの決定に際して、Canonicalのサポートチームは下記の定義を使用します。
重大度レベル1
コア機能が利用できない		 Canonicalは、購入されたサービスレベルに従って、適切なサポートエンジニアまたは開発エンジニアを介して回避策または恒久的な解決策を提供するよう継続的努力します。コア機能が利用可能になった時点で、重大度レベルは新しい適切なサービスレベルに下がります。
重大度レベル2
コア機能が著しく低下している		 Canonicalは、該当する営業時間中、回避策または恒久的な解決策を顧客に提供するよう、一丸となって取り組みます。コア機能の著しい低下が解消された時点で、重大度レベルはレベル3に下がります。
重大度レベル3
標準のサポート要求		 Canonicalは、該当する営業時間中、回避策または恒久的な解決策をできるだけ早く顧客に提供するよう、上位の重大度レベルのケースとのバランスを取りつつ合理的な努力を払います。回避策が提供された場合、Canonicalのサポートエンジニアは、ケースに対する恒久的な解決策を見つけるよう引き続き努力します。
重大度レベル4
緊急性のない要求		 レベル4の要求には、表面上の問題、情報の要求、機能要求などがあります。Canonicalは、いかなる機能要求についても、タイムラインまたは採用の保証は提供しません。Canonicalは、各レベル4ケースをレビューし、それが将来のリリースで検討すべき製品拡張であるか、現行リリースで修正すべき問題であるか、あるいは将来のリリースで修正すべき問題であるかを決定します。情報要求については、サポート範囲の時間中、合理的なレベルの努力でレビューし、それに応答します。Canonicalが適切であると判断した場合は、レベル4の問題を示すケースに応答後、それをクローズできるものとします。

4. ホットフィックス

クリティカルなケースを暫定的に解決するため、Canonicalは、影響を受けるソフトウェア (パッケージなど) のパッチ適用バージョンを提供する場合があります。このようなバージョンは「ホットフィックス」と呼ばれます。Canonicalが提供するホットフィックスは、そのパッチがUbuntu Archivesのソフトウェアのリリースに適用された後、90日間有効です。ただし、上流プロジェクトでパッチが拒否された場合、ホットフィックスのサポートは終了し、ケースはオープンの状態が継続します。

付録3 - Ubuntu Advantageマネジメントエスカレーション

サービスに不満がある場合は、いくつかの方法でその状況をCanonicalの経営陣にエスカレーションできます。

ケース終了後のフィードバック

ケースがクローズされると、Canonicalのサポートの全体的な経験に関するアンケートがケースオーナーにメールで送信されます。すべてのアンケートは経営陣によってレビューされます。

ピアレビューの依頼

Canonicalでは、全ケースの特定の割合に対してピアレビューを行っています。顧客はケースのピアレビューを、ケースコメントまたはサポートポータルに記載の電話番号宛に電話で請求できます。偏見のないエンジニアがケースレビューに任命され、フィードバックを提供します。

マネジメントエスカレーション

緊急でないニーズ：

  • ケース自体の中でマネジメントエスカレーションを要求してください。マネージャーは、ケースをレビューして1営業日以内に応答する必要がある旨の連絡を受け取ります。

緊急なニーズ：

  • Canonicalのサポートおよびテクニカルサービスマネージャーにメール (support-manager@canonical.com) または電話 (+1 514 940 8910) でエスカレーションできます。
  • さらにエスカレーションが必要な場合は、Canonicalのサポートおよびテクニカルサービスディレクターにメール (operations-director@canonical.com) でご連絡ください。
